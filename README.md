# Domestic Utility Worker Management System

A role-based web application built using PHP and MySQL to manage domestic utility services.  
The system connects Customers, Workers, and Administrators to handle service booking, worker assignment, job completion, payment, and feedback through a realistic workflow.

This project was developed as a team-based application, with a strong focus on clean architecture, real-world service flow, and maintainable code.


## Key Highlights

- Role-based system (Customer, Worker, Admin)
- End-to-end service management workflow
- Clean and responsive user interface
- Realistic development and commit history
- Designed for practical, real-world use cases


## Features

### Authentication
- User registration and login
- Secure logout
- Role-based dashboard access

### Customer
- Create service bookings
- Make payment after service completion
- Provide feedback for completed services

###  Worker
- Set and manage availability
- View assigned bookings
- Update booking status after completing work

###  Admin
- View and manage all bookings
- Assign available workers to customer requests

###  User Interface
- Responsive and clean layout
- Modular CSS for authentication, dashboards, forms, tables, and availability pages
- Consistent and user-friendly design


## System Workflow

1. User registers and logs in  
2. Customer creates a service booking  
3. Worker sets availability  
4. Admin assigns worker to the booking  
5. Worker completes the service  
6. Customer makes payment  
7. Customer provides feedback  

This workflow mirrors a real-world domestic service management system.


## Tech Stack

- **Frontend**: HTML5, CSS3  
- **Backend**: PHP  
- **Database**: MySQL  
- **Server**: Apache (XAMPP)
